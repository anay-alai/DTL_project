## Practiced Git Commands

### 1. `git init`
- Initializes a new Git repository in the current directory.
- Creates a hidden `.git` folder to track changes.

### 2. `git clone <url>`
- Copies an existing repository from a remote (like GitHub) to your local machine.

### 3. `git status`
- Shows the current state of the working directory.
- Displays changes staged, unstaged, or untracked.

### 4. `git add <file>`
- Stages changes for commit.
- Example: `git add .` stages all modified files.

### 5. `git commit -m "message"`
- Saves staged changes to the repository history with a descriptive message.

### 6. `git log`
- Displays the commit history.
- Useful for tracking changes and authorship.

### 7. `git branch`
- Lists all branches in the repository.
- `git branch <name>` creates a new branch.

### 8. `git checkout <branch>`
- Switches to the specified branch.
- Can also be used to restore files.

### 9. `git merge <branch>`
- Merges changes from the specified branch into the current branch.

### 10. `git push`
- Uploads local commits to a remote repository (e.g., GitHub).

### 11. `git pull`
- Fetches and integrates changes from a remote repository into the local branch.
